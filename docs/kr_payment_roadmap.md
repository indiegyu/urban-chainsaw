# 한국 결제사 도입 로드맵

> 작성일: 2026-04-09
> 목적: 국내 사용자 대상 디지털 제품/구독 결제 파이프라인 구축

## 1. 결제사 비교 분석

### 1-1. Toss Payments

| 항목 | 내용 |
|---|---|
| **대상 사용자** | 국내 개인/법인 사업자 |
| **수수료** | 카드 2.2~3.3% / 계좌이체 0.5~1% (정책 변동 있음) |
| **정산 주기** | 영업일 기준 D+2 |
| **지원 결제수단** | 카드, 계좌이체, 가상계좌, 토스앱 |

**장점**
- 국내 최고 수준의 개발자 친화적 API 문서 (한국어)
- 웹훅 기반 이벤트 알림 완성도 높음
- 테스트 키/라이브 키 분리 명확
- 빠른 정산 (D+2)

**단점**
- 사업자 등록증 필수 (개인은 제한적)
- 해외 카드 결제 지원 미흡
- 월 최소 거래금액 조건 있을 수 있음

**개발 난이도**: ★★☆☆☆ (쉬움)
**예상 작업량**: 2~3일 (SDK 연동, 웹훅 처리, 결제 성공/실패 핸들링)

---

### 1-2. Kakao Pay

| 항목 | 내용 |
|---|---|
| **대상 사용자** | 카카오톡 사용자 (국내) |
| **수수료** | 협의 (2~3.5% 수준) |
| **정산 주기** | D+3~5 |
| **지원 결제수단** | 카카오페이 잔액, 연결된 카드, 계좌 |

**장점**
- 카카오톡 사용자 기반 압도적 (국내 활성 사용자 4,700만+)
- 모바일 결제 UX 우수
- 구독(정기결제) API 제공

**단점**
- API 문서 품질 불안정 (업데이트 지연)
- 법인 심사 절차 까다로움
- 카카오 생태계 의존도 높음
- 웹 결제창이 팝업 방식 (UX 제약)

**개발 난이도**: ★★★☆☆ (보통)
**예상 작업량**: 3~5일 (API 심사 + 연동 + 정기결제 구현)

---

### 1-3. PayPal (글로벌 대안)

| 항목 | 내용 |
|---|---|
| **대상 사용자** | 해외 + 국내 (글로벌) |
| **수수료** | 3.49% + $0.49 (기준 국제 결제) |
| **정산 주기** | 즉시~3영업일 (PayPal 계좌) |
| **지원 결제수단** | PayPal 잔액, 카드, 은행이체 |

**장점**
- 사업자 등록 없이 개인도 가능 (초기 진입 쉬움)
- 190개국 지원 — 해외 구매자 타겟 시 필수
- REST API 완성도 높고 SDK 다양
- Sandbox 환경 완비

**단점**
- 국내 사용자 인지도 낮음 (한국에서 비주류)
- 수수료 국내 결제사 대비 높음
- 분쟁(Chargeback) 처리 복잡
- 계정 동결 리스크

**개발 난이도**: ★★☆☆☆ (쉬움)
**예상 작업량**: 1~2일 (Checkout SDK 연동)

## 2. 우선순위 및 권장 순서

```
Phase 1 (즉시) → PayPal
Phase 2 (1~2개월) → Toss Payments
Phase 3 (2~4개월) → Kakao Pay
```

**선택 근거:**
- PayPal은 사업자 없이 즉시 시작 가능 → 빠른 수익화 검증
- Toss는 국내 유료 구매자 전환율 최대화 → 핵심 전환 수단
- Kakao는 구독 모델 안정화 후 도입 → 고정 수익 확보

## 3. 테스트 시나리오

### Phase 1 — PayPal
```
1. Sandbox 계정 생성 (buyer/seller 각 1개)
2. 단건 결제 ($9.99 전자책) — 성공/취소/실패 케이스
3. 웹훅 수신 확인 (payment.completed 이벤트)
4. 환불 처리 테스트
5. 라이브 전환 후 실결제 $1 테스트
```

### Phase 2 — Toss Payments
```
1. 테스트 키로 카드 결제 시뮬레이션
2. 가상계좌 발급 → 입금 완료 웹훅
3. 결제 취소/부분취소 케이스
4. 정기결제(빌링) 카드 등록 → 최초결제 → 자동결제 사이클
5. 라이브 키 전환 후 실카드 결제 100원 테스트
```

### Phase 3 — Kakao Pay
```
1. 개발자 계정으로 단건 결제 테스트
2. 정기결제 SID 발급 → 2회차 자동결제
3. 환불 API 테스트
4. 모바일/PC 결제창 분기 처리 검증
```
