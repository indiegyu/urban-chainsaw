# 카카오 i 오픈빌더: 스킬 & 시나리오 상세 가이드

> 오픈빌더 URL: https://i.kakao.com
> 이미 봇이 생성되어 있다고 가정합니다.

---

## Part 1. 스킬(Skill) 등록

스킬 = 우리 서버에 요청을 보내고 응답을 받아오는 연결 설정입니다.

### 1-1. 스킬 메뉴 접근

오픈빌더 화면 왼쪽 사이드바에서:
```
[시나리오] 아이콘
[스킬] 아이콘  ← 여기 클릭
[학습데이터] 아이콘
```

**스킬** 클릭하면 스킬 목록 화면이 나옵니다.

---

### 1-2. 첫 번째 스킬 등록: 나봇_메인메뉴

우측 상단 **+ 스킬 추가** 클릭

**스킬 정보 입력:**

```
스킬 이름: 나봇_메인메뉴
스킬 URL:  https://여기에-ngrok-URL.ngrok-free.app/kakao/webhook/main
```

> ngrok URL은 터미널에서 `ngrok http 8000` 실행 후 나오는
> `https://xxxx.ngrok-free.app` 부분입니다.

**저장** 클릭

저장 후 **테스트** 탭 클릭 → **테스트** 버튼 클릭 → 상태코드 `200` 확인

---

### 1-3. 나머지 5개 스킬도 같은 방법으로 등록

| 스킬 이름 | URL 경로 (ngrok URL + 경로) |
|----------|---------------------------|
| `나봇_키워드추가` | `/kakao/webhook/add-keyword` |
| `나봇_키워드목록` | `/kakao/webhook/list-keywords` |
| `나봇_키워드삭제` | `/kakao/webhook/delete-keyword` |
| `나봇_즉시검색` | `/kakao/webhook/search-now` |
| `나봇_업그레이드` | `/kakao/webhook/upgrade` |

6개 모두 등록 완료 후 스킬 목록:
```
나봇_메인메뉴        ●활성
나봇_키워드추가      ●활성
나봇_키워드목록      ●활성
나봇_키워드삭제      ●활성
나봇_즉시검색        ●활성
나봇_업그레이드      ●활성
```

## Part 2. 시나리오 & 블록 구성

왼쪽 사이드바 맨 위 **시나리오** 아이콘 클릭.

기본 시나리오 `시나리오_1`이 있고, 안에 **폴백 블록**과 **웰컴 블록**이 기본으로 있습니다.

---

### 블록 1: 웰컴 블록 수정

> 기존에 자동 생성된 블록입니다. 새로 만들 필요 없이 수정만 합니다.

좌측 블록 목록에서 **웰컴 블록** 클릭.

**봇 응답** 섹션에서 기존 내용 삭제 후:

**[응답 추가]** → **텍스트** 선택 → 내용 입력:
```
안녕하세요! 나봇 🔍
인터넷에서 나에 대한 얘기를 자동으로 찾아드려요.
(밴드명, 닉네임, 브랜드 등 어떤 키워드든 OK)

아래 버튼을 눌러 시작해보세요!
```

**버튼 추가** (텍스트 응답 편집 화면 하단 **+ 버튼 추가**):
```
버튼 이름: 시작하기
버튼 타입: 블록 연결
연결 블록: 나봇_메인메뉴  ← 아래에서 만들 블록 (지금은 일단 저장, 나중에 연결)
```

**저장** 클릭.

### 블록 3: 나봇_키워드입력

> 키워드 추가 플로우의 1단계. 사용자에게 키워드를 묻습니다.

**+ 블록 추가** 클릭

---

**블록 기본 정보:**
```
블록 이름: 나봇_키워드입력
```

---

**[발화] 섹션:**
```
키워드 추가
추가
등록
키워드 등록
새 키워드
```

---

**[파라미터] 섹션** (블록 편집 화면에서 발화 섹션 아래에 있음):

**+ 파라미터 추가** 클릭:

```
파라미터 이름: term
엔티티: sys.constant
필수 여부: 필수 (토글 ON)
질문 (사용자에게 보낼 메시지):
  모니터링할 키워드를 입력해주세요.
  예) 소음발광, 솜발, 홍길동밴드
```

---

**[봇 응답] 섹션:**

스킬 연결 없이 **텍스트** 응답 선택:
```
'{{term}}'을(를) 검색할 방식을 골라주세요!
```

**버튼 2개 추가:**

버튼 1:
```
버튼 이름: 완전일치 검색 (추천)
버튼 타입: 블록 연결
연결 블록: 나봇_제외어입력  ← 아래에서 만들 블록
전달 파라미터: exact_match = true
```

버튼 2:
```
버튼 이름: 유사검색 포함
버튼 타입: 블록 연결
연결 블록: 나봇_제외어입력
전달 파라미터: exact_match = false
```

> **전달 파라미터 설정 방법:**
> 버튼 편집창 하단 **파라미터 추가** 클릭 → 이름: `exact_match`, 값: `true`

---

**[저장]** 클릭.

## Part 3. 블록 간 연결 마무리

`나봇_메인메뉴` 스킬이 응답에 퀵리플라이를 포함해서 보내므로,

(서버 코드가 blockId를 직접 지정해 보냄)

## Part 4. 블록 ID 확인 및 .env 설정

각 블록을 클릭하면 **블록 상세 화면** 우측 상단에:

```
블록 ID: 67abc123def4567890abcdef  ← 이 값 복사
```

> 또는 블록 목록에서 블록 이름 우측 점 3개(···) 클릭 → "블록 ID 복사"

아래 5개 블록의 ID를 확인해서 기록:

| 블록 이름 | 블록 ID (본인 것으로 교체) |
|----------|--------------------------|
| 나봇_메인메뉴 | `6xxx...` |
| 나봇_제외어입력 | `6xxx...` |
| 나봇_키워드목록 | `6xxx...` |
| 나봇_즉시검색 | `6xxx...` |
| 나봇_업그레이드 | `6xxx...` |

`nabot/.env` 파일에 입력:

```
BLOCK_MAIN=나봇_메인메뉴_블록ID
BLOCK_ADD_KEYWORD=나봇_제외어입력_블록ID
BLOCK_LIST_KEYWORDS=나봇_키워드목록_블록ID
BLOCK_SEARCH_NOW=나봇_즉시검색_블록ID
BLOCK_UPGRADE=나봇_업그레이드_블록ID
```

저장 후 서버 재시작:
```bash
uvicorn main:app --reload --port 8000
```

## Part 5. 오픈빌더 배포 및 테스트

### 배포

오픈빌더 우측 상단 **배포** 버튼 클릭 → **운영채널에 배포** 클릭 → 확인

### 카카오톡 테스트 순서

```
1. 카카오톡 → 채널 검색 → 채널 추가

2. 채팅방 입력: 아무 말이나
   → 웰컴 메시지 + [시작하기] 버튼 확인

3. [시작하기] 클릭 또는 "시작" 입력
   → 현재 플랜(무료), 키워드 0개 메시지 + 버튼 4개 확인

4. [키워드 추가] 클릭
   → "모니터링할 키워드를 입력해주세요" 확인
   
5. "소음발광" 입력
   → "완전일치 검색 / 유사검색 포함" 버튼 확인

6. [완전일치 검색] 클릭
   → "제외할 단어가 있나요?" 확인

7. "없음" 입력
   → "✅ '소음발광' 키워드가 등록됐어요!" 확인

8. [내 키워드 목록] 클릭
   → 등록된 키워드 목록 확인

9. [지금 검색해줘] 클릭
   → 네이버에서 검색 결과 가져와서 표시
   (처음엔 "새로운 언급 없음"이 정상)
```

## 자주 막히는 부분

**Q: 블록 버튼에서 전달 파라미터 설정이 안 보여요**
→ 버튼 타입을 `블록 연결`로 선택해야 파라미터 전달 옵션이 나타납니다.

**Q: 스킬 테스트에서 "연결 실패" 오류**
→ ngrok이 실행 중인지 확인하세요 (`ngrok http 8000`).
→ 서버가 실행 중인지 확인하세요 (`uvicorn main:app --reload`).

**Q: 파라미터가 서버에 안 넘어와요**
→ 스킬 선택 후 나오는 **파라미터 전달** 섹션에서 해당 파라미터에 체크됐는지 확인.

**Q: 배포 후 카카오톡에서 응답이 없어요**
→ 오픈빌더에서 배포 상태 확인. ngrok URL이 만료된 경우 재발급 후 스킬 URL 업데이트 필요.
