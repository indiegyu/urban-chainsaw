# 카카오 i 오픈빌더 연동 가이드

## 시나리오 전체 구조

```
[웰컴 블록] ──→ [메인 메뉴]
                    │
          ┌─────────┼──────────┬──────────┐
          ↓         ↓          ↓          ↓
    [키워드 추가]  [키워드     [즉시     [업그레이드]
    시나리오       목록]       검색]
    │
    ├─ [키워드 입력] (슬롯: term)
    ├─ [검색 방식 선택] (슬롯: exact_match)
    ├─ [제외어 입력] (슬롯: exclude_words)
    └─ [등록 완료] → [메인 메뉴] 복귀
```

---

## 블록별 상세 설정

### [웰컴 블록] (자동 생성, 수정만)
> 카카오톡 채널 추가 시 또는 대화 시작 시 자동 발동

**텍스트 응답** (스킬 연결 없이 그냥 텍스트로):
```
안녕하세요! 나봇 🔍
인터넷에서 나(우리 밴드, 브랜드 등)에 대한 얘기를 자동으로 찾아드려요.

아래 버튼을 눌러 시작해보세요!
```

**버튼** (블록 연결):
- [시작하기] → [메인 메뉴] 블록

---

### [메인 메뉴] 블록
> 스킬: 메인 메뉴 스킬

발화 패턴: `시작`, `처음으로`, `메뉴`, `홈`, `처음`

스킬 응답이 현재 플랜 + 키워드 수를 포함한 텍스트 + 퀵리플라이 4개 반환.

---

### [키워드 추가] 시나리오 (블록 3개로 구성)

#### 블록 1: [키워드 입력]
발화 패턴: `키워드 추가`, `추가`, `등록`

**스킬 없음** — 슬롯만 수집:
- 슬롯 `term` (필수, sys.constant)
  - 안내: "모니터링할 키워드를 입력해주세요. (예: 소음발광, 솜발)"
  - 재입력 안내: "키워드를 다시 입력해주세요."

수집 완료 후 → [검색 방식 선택] 블록으로 자동 이동

#### 블록 2: [검색 방식 선택]
**스킬 없음** — 슬롯 수집:
- 슬롯 `exact_match` (필수, sys.constant)
  - 안내: "검색 방식을 선택해주세요."
  - 버튼형 슬롯:
    - [완전일치 "{{term}}"] → 값: `true`
    - [유사검색 포함] → 값: `false`

수집 완료 후 → [제외어 입력] 블록으로 자동 이동

#### 블록 3: [제외어 입력 + 등록 처리]
**스킬: 키워드 추가 스킬 연결**

슬롯 `exclude_words` (선택, sys.constant):
- 안내: "제외할 단어가 있나요?\n쉼표로 구분해서 입력하거나 '없음'을 입력해주세요.\n예: 고양이,냥이,발바닥"
- 재입력 없음 (스킵 허용)

슬롯 수집 완료 후 → 키워드 추가 스킬 호출
전달 파라미터: `term`, `exact_match`, `exclude_words`

스킬 응답: 등록 완료 메시지 + 메인 메뉴 퀵리플라이

---

### [키워드 목록] 블록
발화 패턴: `내 키워드`, `키워드 목록`, `등록된 키워드`, `목록`
**스킬: 키워드 목록 스킬**

응답에 키워드 ID가 포함됨 → 삭제하려면 "삭제 [번호]" 입력

---

### [키워드 삭제] 블록
발화 패턴: `삭제 *`, `키워드 삭제`
**스킬: 키워드 삭제 스킬**

파라미터 `keyword_id`: 발화에서 숫자 추출 (sys.number 엔티티 사용)

---

### [즉시 검색] 블록
발화 패턴: `지금 검색`, `검색해줘`, `바로 검색`, `검색`
**스킬: 즉시 검색 스킬**

⚠️ API 호출이 포함되어 응답이 3~5초 걸릴 수 있음. 오픈빌더 스킬 타임아웃 5초 이내 처리됨.

---

### [업그레이드] 블록
발화 패턴: `업그레이드`, `유료`, `플랜`, `요금`
**스킬: 업그레이드 스킬**

---

## 전체 흐름

```
카카오 개발자센터 앱
    └─ 카카오톡 채널 연결
    └─ 카카오 i 오픈빌더 연결
            └─ 봇 생성 → 스킬 서버 등록 → 블록 구성
```

## 1단계: 스킬 서버 등록

오픈빌더 > 스킬 > 스킬 추가

| 스킬 이름 | URL |
|----------|-----|
| 메인 메뉴 | `https://your-app.fly.dev/kakao/webhook/main` |
| 키워드 추가 | `https://your-app.fly.dev/kakao/webhook/add-keyword` |
| 키워드 목록 | `https://your-app.fly.dev/kakao/webhook/list-keywords` |
| 키워드 삭제 | `https://your-app.fly.dev/kakao/webhook/delete-keyword` |
| 즉시 검색 | `https://your-app.fly.dev/kakao/webhook/search-now` |
| 업그레이드 | `https://your-app.fly.dev/kakao/webhook/upgrade` |

## 2단계: 블록 구성

### 블록 목록 (시나리오 > 블록 추가)

#### [메인 메뉴] 블록
- 발화 패턴: `시작`, `처음`, `메뉴`, `홈`
- 스킬: 메인 메뉴 스킬 연결
- 파라미터: 없음

#### [키워드 추가 - 입력받기] 블록
- 발화 패턴: `키워드 추가`, `키워드 등록`
- 스킬: 없음 (사용자 발화로 파라미터 수집)
- 슬롯: 아래 2개 슬롯 추가
  - `term` (필수): "모니터링할 키워드를 입력해주세요"
  - `exact_match` (선택): 버튼형 → `완전일치=true`, `유사검색=false`
  - `exclude_words` (선택): "제외할 단어가 있나요? 없으면 '없음' 입력"
- 슬롯 수집 완료 후 → [키워드 추가 처리] 블록 연결

#### [키워드 추가 처리] 블록
- 스킬: 키워드 추가 스킬 연결
- 파라미터 전달: `term`, `exact_match`, `exclude_words`

#### [키워드 목록] 블록
- 발화 패턴: `내 키워드`, `키워드 목록`, `등록된 키워드`
- 스킬: 키워드 목록 스킬 연결

#### [키워드 삭제] 블록
- 발화 패턴: `삭제 *`, `키워드 삭제`
- 스킬: 키워드 삭제 스킬 연결
- 파라미터: `keyword_id` (발화에서 추출)

#### [즉시 검색] 블록
- 발화 패턴: `지금 검색`, `검색해줘`, `바로 검색`
- 스킬: 즉시 검색 스킬 연결

#### [업그레이드] 블록
- 발화 패턴: `업그레이드`, `유료`, `플랜`
- 스킬: 업그레이드 스킬 연결

## 3단계: 블록 ID 확인 후 .env 설정

각 블록 상세 화면 우측 상단에 블록 ID가 표시됩니다.

```
# .env
BLOCK_MAIN=5f3a1b2c3d4e5f6a7b8c9d0e
BLOCK_ADD_KEYWORD=5f3a1b2c3d4e5f6a7b8c9d1f
BLOCK_LIST_KEYWORDS=5f3a1b2c3d4e5f6a7b8c9d2a
BLOCK_SEARCH_NOW=5f3a1b2c3d4e5f6a7b8c9d3b
BLOCK_UPGRADE=5f3a1b2c3d4e5f6a7b8c9d4c
```

Fly.io 배포 환경이라면:
```bash
fly secrets set BLOCK_MAIN=xxx BLOCK_ADD_KEYWORD=xxx ...
```

## 파라미터 슬롯 상세 (키워드 추가)

오픈빌더 슬롯 설정:

| 파라미터명 | 슬롯 타입 | 안내 메시지 |
|-----------|----------|-----------|
| `term` | sys.constant | "모니터링할 키워드를 입력해주세요 (예: 소음발광)" |
| `exact_match` | sys.constant | 버튼: [완전일치 검색] [유사검색 포함] |
| `exclude_words` | sys.constant | "제외할 단어가 있나요? 쉼표로 구분해서 입력하거나 '없음'" |

`exclude_words` 처리: 사용자가 "없음" 입력 시 빈 문자열로 처리 (서버에서 자동 처리).
